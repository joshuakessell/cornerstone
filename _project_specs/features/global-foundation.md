# Feature: global-foundation

**Blocks:** home-page, our-team-page, our-approach-page, our-services-page, contact-and-client-area

## Description
Establish the design system and shared shell that every page depends on: Tailwind theme tokens for the new yellow/gold + navy palette, a typography scale, and shared `<SiteHeader>`, `<SiteFooter>`, and `<PageShell>` components. Also wire up the test infrastructure (Vitest + Testing Library + axe + Playwright) that every other feature will use.

## Acceptance Criteria
1. Tailwind config exposes brand color tokens: `brand-navy-{50..900}` and `brand-gold-{50..900}` (at least 50/100/500/700/900 steps each), replacing any peach/coral references.
2. Brand text tokens (`text-brand-navy`, `text-brand-gold`) and bg tokens (`bg-brand-navy`, `bg-brand-gold`) resolve to values where the 500 step has WCAG AA contrast (≥ 4.5:1) against `white` for gold-on-white is **not** required — but navy-on-white MUST pass AA; gold is used as accent only and must have tooltip/test comment documenting non-text use.
3. A typography scale is defined via `@layer base` with `h1..h6` default sizes that follow a modular scale (ratio ≥ 1.2). Body copy has `text-justify` as the default prose style via a `.prose-body` utility.
4. Main menu link font size is explicitly one step smaller than body (e.g., `text-sm` if body is `text-base`). Test asserts `font-size` computed value.
5. `<SiteHeader>` renders the firm name, a primary nav (Home, Our Team, Our Approach, Our Services, Contact), and **no "Make a Payment" button in the top-right** (that button is consolidated to bottom-right globally — see criterion 6).
6. `<PageShell>` renders a sticky / fixed bottom-right "Make a Payment" button that is an `<a>` to a configurable URL (`NEXT_PUBLIC_PAYMENT_URL` env, with `aria-label="Make a payment"`, `rel="noopener"`, `target="_blank"`). On mobile widths (< 640px) the button remains visible and doesn't overlap content (test via Playwright viewport).
7. `<SiteFooter>` renders the firm name, an address placeholder, and the current year (`new Date().getFullYear()`), with no template boilerplate strings.
8. Test infrastructure present: `vitest.config.ts`, `@testing-library/react`, `@testing-library/jest-dom`, `jsdom` env, `jest-axe` or `axe-core`, and a Playwright config with mobile + desktop viewports. `npm test` runs unit tests; `npm run test:e2e` runs Playwright.
9. Running `npm run build` and `npm run lint` and `npm run typecheck` all pass with zero warnings.
10. An axe-core unit test on `<PageShell><main /></PageShell>` reports **zero violations** with severity `serious` or `critical`.
